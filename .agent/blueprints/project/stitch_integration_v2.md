# 📘 Blueprint: Stitch Integration v2 (Hybrid Bridge)

> **Objective:** Ensure seamless connection between Google Stitch (Cloud) and the local 3-tier CineStack (WSL/Windows).

## 🌍 Rule 1: External Visibility (The Tunnel)
Every hybrid project must support a public tunnel to bridge the Cloud-to-Local gap:
- **Stitch-Mode:** The backend should support an environment flag (e.g., `STITCH_TUNNEL_ACTIVE`) that guides the user to open a secure public tunnel (via ngrok or similar).
- **Public Endpoint:** All MCP configurations for Stitch must use the **Public Tunnel URL** instead of internal IPs (127.0.0.1 or 172.x.x.x).
- **Truth:** `ALWAYS_USE_TUNNEL_FOR_CLOUD_AI = TRUE`.
- **Truth:** Standard CORS is insufficient for Cloud-to-Local; PNA headers are mandatory.
- **Truth (Handshake):** `CORSMiddleware` MUST include `expose_headers=["Access-Control-Allow-Private-Network"]` to work with Google Cloud tools.
- **Truth (Tunnel):** Google Stitch requires a public HTTPS tunnel (ngrok/localtunnel) because it cannot resolve a `.local` or WSL internal IP.

## 📖 Rule 2: Swagger-First Discovery
- **Contract Access:** The `mcp_config.json` must always reference the `/openapi.json` or `/docs` endpoint.
- **Model Awareness:** Stitch must consume the backend schema to understand data structures without needing direct database access.

## 🎨 Rule 3: Asset & UI Resilience
UI components generated or modified by Stitch must adhere to CineStack's surgical standards:
- **Media standard:** Use `CachedNetworkImage` with `BlurHash`.
- **Contrast standard:** Ensure high-contrast overlays (Semi-transparent black) for readability.
- **PNA Safety:** The backend must emit the `Access-Control-Allow-Private-Network: true` header to bypass Chrome's security blocks for local requests.

## 🛠️ Connection Checklist (The Trinity)
1. **Tunnel:** `ngrok http 8001`
2. **Config:** Update `mcp_config.json` -> `"url": "https://<ngrok-id>.ngrok-free.app/openapi.json"`
3. **PNA:** Verify middleware in `main.py` is injecting the PNA header.

## 📱 Rule 4: Mobile & Poco X7 Pro Context
When testing UI generated by Stitch on a mobile device:
- **IP Mismatch:** The device must hit the **Tunnel URL**, not the local IP, if they are on different network segments (e.g., Device on 5G vs. Laptop on WSL).
- **Caching:** Browsers on mobile may aggressively cache PNA preflights; always verify the tunnel status if connection fails.
